## 产品指标仪表板

设计一个全面的产品指标仪表板，包含正确的指标、可视化和警报阈值。

### 上下文

您正在为 **$ARGUMENTS** 设计指标仪表板。

如果用户提供文件（现有仪表板、分析数据、OKR 或战略文档），请先阅读它们。

### 领域上下文

**指标 vs KPI vs NSM**：指标 = 所有可衡量的事物。KPI = 长期跟踪的几个关键量化指标。北极星指标 = 单一的以客户为中心的 KPI，是业务成功的领先指标。

**好指标的 4 个标准**（Ben Yoskovitz，《精益分析》）：(1) 可理解 — 创建通用语言。(2) 可比较 — 随时间变化，而非快照。(3) 比率或速率 — 比整数更具揭示性。(4) 行为改变 — 黄金法则："如果一个指标不会改变你的行为，那它就是一个坏指标。"

**8 种指标类型**：虚荣 vs 可操作（只有可操作指标会改变行为），定性 vs 定量（是什么 vs 为什么 — 你需要两者；永远不要停止与客户交谈），探索性 vs 报告性（探索数据以发现意外洞察），滞后 vs 领先（领先指标启用更快的学习周期，例如客户投诉预测流失）。

**5 个行动步骤**：(1) 根据 4 个好指标标准审核指标。(2) 更新仪表板 — 确保所有关键指标都是好指标。(3) 识别虚荣指标 — 小心使用它们。(4) 分类领先 vs 滞后指标。(5) 选择一个问题并深入研究数据。

有关案例研究和更多详细信息：Ben Yoskovitz 的 [您是否在跟踪正确的指标？](https://www.productcompass.pm/p/are-you-tracking-the-right-metrics)

### 说明

1. **识别指标框架** — 将指标组织成层次：

   **北极星指标**：最能捕捉核心价值交付的单一指标

   **输入指标**（3-5 个）：驱动北极星的杠杆

   **健康指标**：确保整体产品健康的护栏

   **业务指标**：收入、成本和单位经济效益

2. **为每个指标定义**：

   | 指标 | 定义 | 数据源 | 可视化 | 目标 | 警报阈值 |
   |---|---|---|---|---|---|
   | [名称] | [精确计算：分子/分母，时间窗口] | [数据来源] | [折线图 / 柱状图 / 数字 / 漏斗图] | [目标值] | [何时触发警报] |

3. **设计仪表板布局**：

   ```
   ┌─────────────────────────────────────────────┐
   │  北极星：[指标] — [当前值]                 │
   │  趋势：[↑/↓ X% vs 上一期间]                │
   ├──────────────────┬──────────────────────────┤
   │  输入指标 1      │  输入指标 2              │
   │  [迷你图]        │  [迷你图]                │
   ├──────────────────┼──────────────────────────┤
   │  输入指标 3      │  输入指标 4              │
   │  [迷你图]        │  [迷你图]                │
   ├──────────────────┴──────────────────────────┤
   │  健康：[延迟] [错误率] [NPS]              │
   ├─────────────────────────────────────────────┤
   │  业务：[MRR] [CAC] [LTV] [流失率]          │
   └─────────────────────────────────────────────┘
   ```

4. **设置审查节奏**：
   - **每日**：运营健康（错误、延迟、关键流程）
   - **每周**：输入指标和参与趋势
   - **每月**：北极星、业务指标、OKR 进展
   - **每季度**：战略审查和指标重新校准

5. **定义警报**：
   - 什么阈值触发调查？
   - 谁会收到警报以及通过什么渠道？
   - 预期的响应时间是多少？

6. **根据用户上下文推荐工具**：
   - Amplitude、Mixpanel、PostHog 用于产品分析
   - Looker、Metabase、Mode 用于基于 SQL 的仪表板
   - Datadog、Grafana 用于运营健康

逐步思考。将仪表板规范保存为 markdown 文档。
