## 定价策略

设计基于价值传递、竞争定位和支付意愿的定价策略。

### 背景

您正在为 **$ARGUMENTS** 开发定价策略。

如果用户提供文件（竞争对手定价、调查数据、财务模型或使用数据），请先阅读它们。如有需要，使用网络搜索研究竞争对手定价。

### 说明

1. **了解传递的价值**：
   - 核心价值主张是什么？
   - 客户的替代方案是什么（及其成本）？
   - 产品提供哪些可量化的成果？（节省时间、增加收入、降低成本）
   - 基于该价值，客户的支付意愿是多少？

2. **评估定价模型** — 推荐最佳拟合：

   | 模型 | 最适合 | 示例 |
   |---|---|---|
   | **固定费率** | 简单产品，可预测成本 | Basecamp（$99/月固定） |
   | **每席位** | 协作工具，团队产品 | Slack, Figma |
   | **基于使用** | 基础设施，API 产品 | AWS, Twilio |
   | **分层** | 具有不同用户细分的产品 | 大多数 SaaS（免费/专业/企业） |
   | **免费增值** | 具有病毒/网络效应的产品 | Spotify, Notion |
   | **免费增值 + 使用** | 平台产品 | Vercel, OpenAI API |
   | **基于价值** | 高影响力企业工具 | Salesforce, Palantir |

3. **分析竞争定价**：
   - 映射竞争对手定价层级及包含内容
   - 确定您的产品定位（高端、中端市场、预算）
   - 寻找定价差距或机会
   - 注意任何行业定价惯例

4. **设计定价结构**：
   - **层级**：定义 2-4 个具有明确差异化的层级
   - **功能限制**：哪些功能进入哪个层级？（使用价值指标，而非任意限制）
   - **价值指标**：您基于什么单位收费？（用户、事件、存储、API 调用）
   - **锚定定价**：将最受欢迎的层级设置为明显的选择
   - **年度折扣**：通常比月度定价低 15-20%

5. **估计价格敏感性**：
   - Van Westendorp 价格敏感性计（如果有调查数据）：
     - 太便宜 → 质量担忧
     - 便宜 → 良好价值
     - 昂贵 → 开始犹豫
     - 太贵 → 不会购买
   - 或者，基于竞争对手定价和传递的价值进行估计

6. **规划定价实验**：
   - A/B 测试定价页面（不同价格点、层级名称、功能捆绑）
   - 创始人主导的销售对话测试支付意愿
   - 具有不同价格锚点的着陆页测试
   - 按价格点进行转化率的队列分析

7. **输出定价建议**：
   ```
   推荐模型：[模型类型]
   价值指标：[您基于什么收费]

   | 层级 | 价格 | 目标细分 | 关键功能 | 定位 |
   |---|---|---|---|---|

   关键假设：
   - [假设] → [如何测试]

   风险：
   - [风险] → [缓解措施]
   ```

逐步思考。保存为 markdown。标记任何在启动前需要验证的假设。
