# 队列分析与留存探索

## 目的
通过队列分析用户参与和留存模式，以识别用户行为、功能采用和长期参与的趋势。将定量洞察与定性研究建议相结合。

## 工作原理

### 步骤 1：读取和验证数据
- 接受包含用户队列信息的 CSV、Excel 或 JSON 数据文件
- 验证数据结构：队列标识符、时间段、参与指标
- 检查缺失值和数据质量问题
- 汇总关键统计数据（队列大小、日期范围、可用指标）

### 步骤 2：生成定量分析
- 计算队列留存率和参与趋势
- 识别留存曲线、下降模式和异常
- 计算跨队列的功能采用率
- 计算月环比或期环比变化
- 如果请求，使用 pandas 和 numpy 生成 Python 分析脚本

### 步骤 3：创建可视化
- 生成留存热图（队列 vs. 时间段）
- 创建显示队列进展的折线图
- 构建功能采用比较图
- 可视化下降点和参与趋势
- 输出为交互式图表或静态图像

### 步骤 4：识别洞察和模式
- 发现一个或多个显著模式：
  - 特定队列的早期流失
  - 后期参与变化
  - 功能采用集群
  - 季节性或时间趋势
- 突出显示令人惊讶的发现和偏差
- 比较队列性能以建立基线

### 步骤 5：建议后续研究
- 推荐定性研究方法：
  - 对流失用户的有针对性用户访谈
  - 对参与队列的功能使用调查
  - 关键交互模式的会话回放
  - 高留存与低留存队列的胜负分析
- 设计后续定量研究
- 建议 A/B 测试或功能实验

## 使用示例

**示例 1：上传 CSV 数据**
```
上传 cohort_engagement.csv，包含列：cohort_month, weeks_active,
user_id, feature_x_usage, engagement_score

请求："分析留存模式并确定为什么 2025 年第四季度队列
表现不如第三季度"
```

**示例 2：描述数据格式**
```
"我有 2025 年 1 月至 12 月的月度用户队列。每行显示：
队列日期、用户 ID、购买频率和支持工单。
分析哪些队列显示最佳长期留存。"
```

**示例 3：功能采用分析**
```
上传包含队列采用数据的 feature_usage.xlsx。

请求："比较我们新功能在各队列中的采用曲线。
哪些队列采用最快？有任何模式吗？"
```

## 关键功能

- **数据读取**：导入 CSV、Excel、JSON、SQL 查询结果
- **留存分析**：计算和可视化随时间的留存率
- **队列比较**：比较队列组之间的指标
- **异常检测**：标记不寻常的模式或下降
- **Python 脚本**：生成为持续分析的可重用分析代码
- **可视化**：创建热图、图表和交互式仪表板
- **研究设计**：建议有针对性的后续研究和访谈方法
- **统计摘要**：提供定量指标和相关性分析

## 最佳结果提示

1. **包含时间维度**：提供多个时间段的数据
2. **明确定义队列**：使队列分组明确（注册月份、功能发布日期等）
3. **提供上下文**：解释期间的产品变更、发布或事件
4. **多个指标**：包括留存、参与、功能使用、收入等
5. **足够的数据**：至少 3-4 个队列以进行有意义的模式识别
6. **请求特定输出**：要求可视化、Python 脚本或研究建议

## 输出格式

您将收到：
- **数据摘要**：队列概述和数据质量评估
- **定量发现**：关键指标、留存率和趋势分析
- **可视化**：显示留存曲线、采用模式的图表
- **模式识别**：数据中的 2-3 个显著洞察
- **研究建议**：具体的定性和定量后续行动
- **分析脚本**（如果请求）：用于可重复分析的 Python 代码
- **后续步骤**：基于发现的优先行动
