# SQL 查询生成器

## 目的

将自然语言需求转换为跨多个数据库平台的优化 SQL 查询。此技能帮助产品经理、分析师和工程师生成准确的查询，无需手动语法工作。

## 工作原理

### 步骤 1：了解您的数据库模式

- 如果您提供模式文件（SQL、文档或图表描述），我将读取并分析它
- 提取表名、列定义、数据类型和关系
- 识别主键、外键和索引策略

### 步骤 2：处理您的请求

- 明确您需要检索或分析的确切数据
- 确认 SQL 方言（BigQuery、PostgreSQL、MySQL、Snowflake 等）
- 询问任何其他要求（过滤器、聚合、排序）

### 步骤 3：生成优化查询

- 编写利用数据库结构的高效 SQL
- 包含解释复杂逻辑的注释
-
 为大型数据集添加性能考虑
- 提供适用的替代方法

### 步骤 4：解释和测试
- 用简单的英语解释查询逻辑
- 建议如何测试或验证结果
- 提供性能优化提示
- 如果需要，生成测试脚本或示例数据

## 使用示例

**示例 1：从模式文件查询**
```
上传您的 database_schema.sql 文件并说：
"生成一个查询，查找在过去 30 天内注册且至少有 5 个活跃会话的用户"
```

**示例 2：从图表描述查询**
```
"这是我的数据库：Users 表（id, email, created_at），Sessions 表
(id, user_id, timestamp, duration)。生成 2026 年 1 月每个用户的平均会话
持续时间的查询。"
```

**示例 3：复杂分析查询**
```
"创建一个 BigQuery 查询，按地区和客户层级分析我们的收入，
包括同比增长率。"
```

## 关键功能

- **多方言支持**：适用于 BigQuery、PostgreSQL、MySQL、Snowflake、SQL Server
- **文件读取**：读取模式文件、SQL 转储和数据文档
- **查询优化**：建议索引、分区和性能改进
- **解释**：分解查询以供学习和文档记录
- **测试**：可以生成测试查询和示例数据脚本
- **脚本执行**：为您的数据库创建可执行 SQL 脚本

## 最佳结果提示

1. **提供上下文**：分享您的数据库模式或结构
2. **具体明确**：清楚描述您需要的数据和任何过滤器
3. **提及数据库**：指定您使用的 SQL 方言
4. **包含约束**：提及数据量、时间范围和性能需求
5. **请求格式**：如果需要特定输出，询问查询结果格式

## 输出格式

您将收到：
- **SQL 查询**：带有注释的生产就绪 SQL 代码
- **解释**：查询的作用和工作原理
- **性能说明**：优化提示和考虑因素
- **测试脚本**（如果请求）：示例数据和验证查询
