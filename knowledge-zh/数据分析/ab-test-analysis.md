## A/B 测试分析

使用统计严谨性评估 A/B 测试结果，并将发现转化为清晰的产品决策。

### 上下文

您正在分析 **$ARGUMENTS** 的 A/B 测试结果。

如果用户提供数据文件（CSV、Excel 或分析导出），请直接读取并分析它们。必要时生成 Python 脚本进行统计计算。

### 说明

1. **理解实验**：
   - 假设是什么？
   - 更改了什么（变体）？
   - 主要指标是什么？有任何护栏指标吗？
   - 测试运行了多长时间？
   - 流量分配是多少？

2. **验证测试设置**：
   - **样本量**：样本是否足够大以检测预期效果大小？
     - 使用公式：n = (Z²α/2 × 2 × p × (1-p)) / MDE²
     - 如果测试功效不足（<80%功效），请标记
   - **持续时间**：测试是否至少运行了 1-2 个完整的业务周期？
   - **随机化**：是否有样本比例不匹配（SRM）的证据？
   - **新颖性/首因效应**：是否有足够的时间消除初始行为变化？

3. **计算统计显著性**：
   - 对照组和变体的**转化率**
   - **相对提升**：(变体 - 对照组) / 对照组 × 100
   - **p 值**：使用双尾 z 检验或卡方检验
   - **置信区间**：差异的 95% CI
   - **统计显著性**：p < 0.05？
   - **实际显著性**：提升对业务是否有意义？

   如果用户提供原始数据，请生成并运行 Python 脚本来计算这些值。

4. **检查护栏指标**：
   - 任何护栏指标（收入、参与度、页面加载时间）是否下降？
   - 主要指标获胜但护栏指标下降可能不是真正的胜利

5. **解释结果**：

   | 结果 | 建议 |
   |---|---|
   | 显著正向提升，无护栏问题 | **发布** — 100% 推出 |
   | 显著正向提升，有护栏问题 | **调查** — 在发布前了解权衡 |
   | 不显著，正向趋势 | **延长测试** — 需要更多数据或更大效果 |
   | 不显著，持平 | **停止测试** — 未检测到有意义的差异 |
   | 显著负向提升 | **不要发布** — 恢复到对照组，分析原因 |

6. **提供分析摘要**：
   ```
   ## A/B 测试结果：[测试名称]

   **假设**：[我们的预期]
   **持续时间**：[X 天] | **样本**：[N 对照组 / M 变体]

   | 指标 | 对照组 | 变体 | 提升 | p 值 | 显著？ |
   |---|---|---|---|---|---|
   | [主要指标] | X% | Y% | +Z% | 0.0X | 是/否 |
   | [护栏指标] | ... | ... | ... | ... | ... |

   **建议**：[发布 / 延长 / 停止 / 调查]
   **理由**：[为什么]
   **下一步**：[做什么]
   ```

请逐步思考。保存为 Markdown。如果提供原始数据，生成 Python 脚本进行计算。
