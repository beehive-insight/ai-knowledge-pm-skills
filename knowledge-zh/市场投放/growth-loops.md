# 增长循环

## 概述
识别并设计创造可持续牵引力的增长循环（飞轮）。此技能评估五种经过验证的增长循环机制，以减少对付费获取的依赖并构建产品主导的增长。

## 适用场景
- 为产品设计增长机制
- 建立可持续的病毒式或推荐牵引力
- 减少对付费获取的依赖
- 分析竞争对手的增长策略
- 优化产品以实现产品主导的增长

## 5 种增长循环类型

### 1. 病毒式循环
用户创建的产品内容在外部平台上共享，将新用户带回产品。
- **机制**：用户在产品中创建内容 → 在社交/外部平台上共享 → 新用户发现并注册
- **示例**：Figma 设计作为链接共享，Loom 视频在电子邮件中共享
- **优势**：如果内容本身可共享，用户获取呈指数增长
- **挑战**：需要高度可共享的输出和强烈的共享动机

### 2. 使用循环
用户在产品内创建内容或价值，然后共享，邀请新用户或推动重新参与。
- **机制**：用户创建 → 分享创作 → 其他人消费 → 成为活跃用户
- **示例**：Twitter 线程、Medium 文章、公开共享的 Notion 模板
- **优势**：增长直接与产品使用和网络效应相关
- **挑战**：需要内容创建的摩擦非常低

### 3. 协作循环
用户邀请同事在产品内共同创建或协作，在组织内扩大用户基础。
- **机制**：用户创建 → 邀请同事协作 → 同事发现产品价值
- **示例**：Google Docs 邀请、Figma 团队项目、Slack 频道
- **优势**：深度组织渗透和高留存
- **挑战**：最适合协作/团队型产品

### 4. 用户生成循环
用户通过其他用户的创作发现新内容或功能，然后创建并分享自己的内容。
- **机制**：用户发现内容 → 创建类似内容 → 分享创作 → 其他人发现
- **示例**：TikTok、Pinterest、YouTube 趋势驱动创作者参与
- **优势**：创建内容飞轮和网络效应
- **挑战**：需要足够质量的内容来维持

### 5. 推荐循环
用户邀请其他潜在用户以换取奖励、激励或社会认可。
- **机制**：用户推荐 → 被推荐用户加入 → 推荐者获得奖励 → 分享更多推荐
- **示例**：Dropbox 推荐奖励、Uber 乘客推荐、PayPal 注册奖励
- **优势**：直接激励获取；易于衡量 ROI
- **挑战**：需要有价值的激励，同时不侵蚀单位经济

## 工作原理

### 步骤 1：定义产品价值
澄清用户体验的核心价值：
- 用户在您的产品中采取的主要行动
- 每个用户行动创造的价值
- 存在的网络效应（如有）
- 体验中的摩擦点

### 步骤 2：评估循环适合度
评估哪些增长循环与您的产品对齐：
- 产品类型（协作、基于内容、实用工具等）
- 目标用户行为和共享习惯
- 已经存在的网络效应
- 现有用户基础和参与度

### 步骤 3：设计循环机制
创建具体的循环实现：
- 启动共享或邀请的触发器
- 参与的激励（内在或外在）
- 分享机制的便捷性
- 从邀请到激活的转化率
- 每个用户循环重复的频率

### 步骤 4：计算循环系数
估计增长速度：
- 每个周期每个用户的邀请/分享数
- 邀请转化为新用户的转化率
- 每个周期的净新用户数
- 每个周期迭代的时间

### 步骤 5：构建循环
首先实现最高杠杆的循环：
- 从最适合您产品的循环开始
- 优化消息传递和摩擦
- 衡量循环指标和转化率
- 随着时间的推移复合结果

## 输入格式
使用 $ARGUMENTS 传递：
- 产品描述和主要用户行动
- 目标用户人口统计和行为
- 现有的共享/协作功能
- 当前的增长渠道和指标
- 约束或机会

## 输出
增长循环分析，包括：
- 为您的产品对所有 5 种循环类型的排名评估
- 推荐的主要增长循环及其实施计划
- 随时间分层的次要循环
- 关键指标和衡量框架
- 30-60-90 天实施路线图
- 潜在的循环系数和增长预测

## 框架
基于 Ognjen Bošković 的增长循环研究。专注于通过内置的、产品原生的共享和协作机制实现复合用户获取。

## 提示
- 从一个循环开始，掌握它后再增加复杂性
- 病毒式循环复合最快，但需要时间构建
- 协作循环创造最强的留存和 LTV
- 在优化阶段每周衡量循环健康度
- 一旦大规模运行，组合循环以获得乘法效应
