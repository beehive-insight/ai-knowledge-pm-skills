## 客户旅程地图

映射从认知到推荐的端到端客户体验，识别每个阶段的情绪、痛点和改进机会。

### 上下文

您正在为 **$ARGUMENTS** 创建客户旅程地图。

如果用户提供文件（访谈记录、调查数据、分析、支持工单或现有旅程地图），请先阅读它们。如果提供 URL，请使用网络搜索了解产品。

### 说明

1. **定义角色**：谁在旅行这个旅程？使用具有 JTBD 的特定角色，而非通用用户。

2. **映射旅程阶段**（根据产品调整）：

   | 阶段 | 描述 |
   |---|---|
   | **认知** | 他们如何首次了解产品？ |
   | **考虑** | 他们评估什么？他们比较什么替代方案？ |
   | **获取** | 他们如何注册或购买？ |
   | **入职** | 产品的首次体验 — 价值实现时间 |
   | **参与** | 定期使用 — 建立习惯 |
   | **保留** | 什么让他们回来？什么可能导致流失？ |
   | **推荐** | 他们何时以及为什么向他人推荐产品？ |

3. **为每个阶段记录**：

   - **接触点**：用户与产品、品牌或团队互动的地方（网站、电子邮件、应用内、支持、社交媒体）
   - **用户行动**：他们在这个阶段做什么
   - **想法和问题**：他们在想什么（"这值得我的时间吗？" "我如何...？"）
   - **情绪**：他们的感受（兴奋、困惑、沮丧、高兴）— 用 scale 或 emoji 指标评分
   - **痛点**：摩擦、困惑、放弃风险
   - **机会**：如何在这一点改善体验

4. **识别关键时刻**：
   - **Aha 时刻**：用户首次体验核心价值的时候
   - **关键时刻**：他们承诺或放弃的决策点
   - **流失触发点**：用户最常放弃的地方

5. **创建旅程地图表**：

   | 阶段 | 接触点 | 用户行动 | 情绪 | 痛点 | 机会 |
   |---|---|---|---|---|---|

6. **推荐优先改进**：
   - 哪些痛点对转化或保留的影响最大？
   - 哪些快速胜利可以立即改善体验？
   - 哪些需要更深入的投资但回报最大？

请逐步思考。保存为 Markdown 文档。对于可视化旅程地图，建议用户使用此分析作为基础在 Miro 或 FigJam 中创建一个。
