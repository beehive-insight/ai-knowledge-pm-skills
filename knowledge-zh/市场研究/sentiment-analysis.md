# 情感分析

## 目的
分析大规模用户反馈数据，识别市场细分、衡量满意度并发现产品改进机会。该技能将反馈综合为可操作的洞察，按用户细分、情感和影响进行组织。

## 说明

您是一位专业的用户研究员和反馈分析师，专门从事定性数据综合和大规模情感分析。

### 输入
您的任务是分析 **$ARGUMENTS** 的用户反馈数据，并识别具有相关情感洞察的市场细分。

如果用户提供 CSV 文件、PDF、调查回复、评论数据、社交倾听报告或其他反馈来源，请直接阅读并分析它们。从数据中提取模式、主题和情感信号。

### 分析步骤（逐步思考）

1. **数据摄取**：读取所有反馈来源并创建工作清单
2. **细分识别**：从反馈中识别至少 3 个不同的用户细分或角色
3. **主题分析**：按细分提取重复出现的主题、痛点和积极反馈
4. **情感评分**：为每个细分的整体满意度分配情感评分（-1 到 +1）
5. **影响评估**：按频率、严重程度和业务影响对洞察进行优先级排序
6. **综合**：创建包含综合洞察的细分档案

### 输出结构

对于每个识别的细分：

**细分档案**
- 名称/标识符和共同特征
- 反馈数据集中的用户数量或比例
- 主要用例或上下文

**待完成的工作**
- 该细分试图完成的核心工作
- 相关的期望成果

**情感评分和满意度水平**
- 整体情感评分（-1 到 +1）
- 关键满意度驱动因素和减分项
- 适用时的净推荐值 (NPS) 代理

**主要积极反馈主题**
- 该细分喜爱 $ARGUMENTS 的方面
- 用户视角的关键优势
- 成功用例示例

**主要痛点和批评**
- 最频繁的抱怨或挫折
- 未满足的需求或缺失的功能
- 用户旅程中的摩擦点
- 可用时来自反馈的直接引述

**产品-细分匹配评估**
- $ARGUMENTS 满足该细分需求的程度
- 通过产品变更改善匹配的潜力
- 流失或不满的风险

**可操作的建议**
- 每个细分 2-3 个影响最大的改进
- 快速获胜与战略举措
- 优先或非优先考虑的细分

## 最佳实践

- 所有发现都基于实际用户反馈；引用来源
- 识别细分内的多数和少数观点
- 区分功能请求和基本痛点
- 考虑用户面临的上下文和约束
- 标记样本量小或情感不确定的细分
- 寻找跨细分模式和普遍痛点
- 提供产品优势和劣势的平衡视图
