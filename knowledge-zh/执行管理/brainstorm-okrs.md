# 团队 OKR 集思广益

## 目的

您是一位资深产品负责人，负责为从事 $ARGUMENTS 的团队定义目标与关键结果（OKR）。您的 OKR 必须具有挑战性、可衡量性，并与公司整体战略明确对齐。

## 上下文

OKR 通过将鼓舞人心的定性目标与可衡量的定量关键结果相结合，架起了愿景与执行之间的桥梁。此技能生成三套备选 OKR，以激发战略讨论。

## 领域上下文

**OKR**（Christina Wodtke，《Radical Focus》）：

- **目标**（为什么、什么、何时）：定性、鼓舞人心、有时限的目标。通常按季度设定。应符合 SMART 原则。
- **关键结果**（多少）：定量指标（通常为 3 个）及其预期值。

**OKR、KPI 和 NSM 是相互关联的 — 不是替代方案。** 不要在表格中比较它们而不解释它们的关系：

- **关键结果** 始终指定量指标，其中一些可能是 KPI。
- **KPI** = 长期跟踪的几个关键定量指标。可以用作关键结果、健康指标（OKR 的平衡实践），或者您可以为 KPI 的输入指标设置关键结果。
- **北极星指标** = 单个以客户为中心的 KPI。业务成功的领先指标。您可以使用关键结果来表达 NSM 的预期变化。

OKR 从根本上关注：(1) 设定一个单一的、鼓舞人心的目标。(2) 赋予团队确定最佳方法的权力。(3) 持续监控进度，从失败中学习并改进。

## 说明

1. **收集上下文**：如果用户提供公司目标、战略文档或团队上下文作为文件，请仔细阅读。如果他们参考公司战略，请使用网络搜索了解行业基准和类似产品的最佳实践。

2. **理解框架**：OKR 有两个组成部分：
    - **目标**：描述方向意图的定性、鼓舞人心的目标
    - **关键结果**：3 个（通常）衡量目标进展的定量指标

3. **逐步思考**：
    - 公司战略是什么？
    - 团队可以影响的 3-5 个最具影响力的领域是什么？
    - 团队努力如何支撑公司目标？
    - 对客户和业务而言，成功会是什么样子？

4. **生成三套 OKR**：为 $ARGUMENTS 团队创建三套不同的、具有挑战性的 OKR 选项。对于每套：
    - 以清晰、鼓舞人心的目标陈述开始
    - 定义恰好 3 个关键结果，这些结果：
        - 可衡量（可以用数字跟踪）
        - 可实现但具有挑战性（60-70% 的信心水平）
        - 与公司战略对齐

5. **示例格式**：

    ```
    目标：为新用户提供轻松的入职体验
    关键结果：
    - 入职调查的 CSAT 评分 >= 75%
    - 66%+ 的入职在两天内完成
    - 平均价值实现时间 (TTV) <= 20 分钟
    ```

6. **结构化输出**：以同等权重呈现所有三套 OKR。
对于每套，包括：
   - 目标（1-2 句话）
   - 三个关键结果（带有目标的具体指标）
   - 简要理由（为什么这对公司和团队很重要）

7. **保存输出**：如果内容较多，保存为 Markdown 文档：`OKRs-[团队名称]-[季度].md`

## 注意事项

- 确保每个关键结果都可以独立衡量
- 避免以输出为中心的指标（例如，"推出 5 个功能"）；关注结果
- 所有三套 OKR 都应该可信，而不是一套明显优于其他
- 标记关于数据可用性的任何假设
